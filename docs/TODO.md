## [P1] 跨会话记忆的被动召回

**发现时间**：2026-04-23（线上 9 轮深度对话测试）
**关联模块**：`lib/agent/` · `convex/memories.ts` · `app/api/chat/route.ts`

### 现象
在新会话中提到与历史记忆强相关的主题（例：用户在前一次会话讲述过父亲离世；新会话里描述"梦到在老家找一个找不到的人，醒来哭很久"），AI 并未主动召回记忆，反而像陌生人一样重新询问"你父亲还在身边吗？"。直到用户明确说"我们上次聊过"，记忆才被精准拉取。

### 本质
当前记忆召回链路是**懒加载（tool-call-driven）**——由 LLM 在推理中自行决定是否调用 memory 工具。模型倾向于在证据足够明确时才触发工具调用，而像"梦到某人"这类隐喻性信号未被判定为足够强的召回信号。结果是：产品承诺的"永恒记忆"在用户主观体验上表现为"每次都要重新介绍自己"。

### 建议
在每次 `/api/chat` 请求进入 LLM 之前，**被动预取一次语义相似的记忆**（embedding top-k，阈值过滤），作为 system message 上下文注入，与现有的 tool call 记忆机制并行：

- 主动层（embedding recall）：覆盖"应该记得"的常识性关联
- 工具层（memory tool）：保留模型主动深挖的能力

两层互补后，长老人设的"记得你"才真正落地。

### 验收
- 新会话里提及旧事件的隐喻描述（梦、意象、侧面提问），AI 能在首轮回复中自然引用记忆
- 无关话题不会被错误召回（false positive < 10%）
- 首 token 延迟增加 ≤ 200ms
